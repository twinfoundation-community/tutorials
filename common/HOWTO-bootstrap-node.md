# Steps to bootstrap a Node

1. Create a Decentralized Identity for the Node

```sh
./twin-node.sh identity-create
```

1. Add key to the Node's DID for auth token signing

```sh
./twin-node.sh vault-key-create --identity="did:entity-storage:0x0565892dfbcdcbdf9e93a2f332c68cc40935ba8d50d720bc38ed22bedc72071a" --key-id="auth-signing" --key-type="Ed25519"
```

1. Set Node Identity

```sh
/twin-node.sh node-set-identity --identity="did:entity-storage:0x0565892dfbcdcbdf9e93a2f332c68cc40935ba8d50d720bc38ed22bedc72071a"
```

1. Create organization identity

```sh
./twin-node.sh identity-create
```

1. Add verification methods to organization's identity

This verification method is for authentication as data provider or consumer in Dataspace exchanges.

```sh
./twin-node.sh identity-verification-method-create --identity="did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed" --verification-method-id="trust-assertion"
```

This verification method is for attestation assertions, for instance when tokenizing a document as an [MLETR](https://uncitral.un.org/en/texts/ecommerce/modellaw/electronic_transferable_records) record.

```sh
./twin-node.sh identity-verification-method-create --identity="did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed" --verification-method-id="attestation-assertion"
```

This verification method is for immutable proofs generated by an organization that enable data verifiability.

```sh
./twin-node.sh identity-verification-method-create --identity="did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed" --verification-method-id="immutable-proof-assertion"
```

Add an encryption key for blob storage at rest, to encrypt documents.

```sh
./twin-node.sh vault-key-create --identity="did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed" --key-id="blob-encryption" --key-type="ChaCha20Poly1305"
```

1. Create a tenant

```sh
./twin-node.sh tenant-create
```

```sh

Organization ID: did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed
Label: Tenant
Public Origin:
```

1. Create an Admin User

```sh
./twin-node.sh identity-create
```

Although not strictly necessary a verification method could be added for the user.

```sh
./twin-node.sh identity-verification-method-create --identity="did:entity-storage:0x27fd6b9c9951165fae9e1160cee946308f6776d3abb72a61cf55853321743417" --verification-method-id="sign-0"
```

1. Create User with login and pass

This login and pass will allow to generate authentication tokens for regular Node services (not Dataexchange operations).

```sh
twin-node.sh user-create --user-identity="did:entity-storage:0x27fd6b9c9951165fae9e1160cee946308f6776d3abb72a61cf55853321743417" --organization-identity="did:entity-storage:0x49686fbc29d48ff89813cd0743874507002cc4744aa515e8298e5b15a0eaf9ed" --email="jcantera@example.com" --password="yM5?NgPPAio+TmWx" --scope="tenant-admin" --tenant-id="019eef73d629745a99083dc7faaae5d8"
```


## Using Docker Compose

All the instructions above can be performed with `docker compose`, as a way example creating an identity would be performed as follows:

```sh
docker compose run --rm twin-node sh -c 'node src/index.js identity-create --load-env=.env.local.bootstrap'
```
